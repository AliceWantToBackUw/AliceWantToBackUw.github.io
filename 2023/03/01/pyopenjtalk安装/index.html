# 💢报错

​ 安装 pyopenjtalk 始终报错
< img src="https://cdn.staticaly.com/gh/AliceWantToBackUw/blog-img@main/PicGo/202303012018645.png">
​ 没有找到 cmake 的模块。

# 💘解决

​ 先去 github 查看该包 cmake がインストールしたのに "ModuleNotFoundError: No module named 'cmake'"・Issue #33・r9y9/pyopenjtalk (github.com)<img src="https://cdn.staticaly.com/gh/AliceWantToBackUw/blog-img@main/PicGo/202303012022638.png">
​ 可以发现有人已经提出来了，给出了他的修改方案
< img src="https://cdn.staticaly.com/gh/AliceWantToBackUw/blog-img@main/PicGo/202303012023005.png">

  • 将 python 编译器添加到系统环境变量中，
  • 注意是将 Visual Studio 的 CMake\bin 添加到系统环境变量中！！
    <img src="https://cdn.staticaly.com/gh/AliceWantToBackUw/blog-img@main/PicGo/202303012026928.png">
    原因：可能是 README.md 提到的 需要 C/C++ 编译器扩展
    < img src="https://cdn.staticaly.com/gh/AliceWantToBackUw/blog-img@main/PicGo/202303012028187.png">

​ 至此，可以 pip install pyopenjtalk 安装成功了。
​ 但是，部分人还是无法安装成功，请 pip list 检查一下是否安装了 cmake 包。
先 pip uninstall cmake 卸载 cmake ，再安装！
